🔄 典型业务流程

📖 博客查询流程

1
客户端发送 GET /api/blogs 请求
2
BlogController 接收请求，调用 BlogFileService
3
BlogFileService 读取文件系统中的 Markdown 文件
4
解析文件内容，创建 Blog 对象列表
5
封装为 Message 对象，返回 JSON 响应

💬 LLM 对话流程

1
客户端发送 POST /api/llm/stream 请求
2
LLMController 通过工厂创建 LLM 实例
3
LLM 服务调用第三方 API
4
使用 SSE 向客户端返回流式响应

✨ 架构优势

🎯 设计优势

  • 分层清晰，职责明确，易于维护
  • 工厂模式支持多 LLM 平台扩展
  • 基于文件系统，部署简单
  • RESTful API 设计，前后端分离
  • 流式响应提供更好的用户体验
  • 配置化管理，灵活可配置

🌐 API 接口概览

📝 博客 API

  • GET /api/blogs - 获取所有博客
  • POST /api/blogs - 保存博客
  • DELETE /api/blogs/{title} - 删除博客
  • POST /api/blogs/search - 搜索博客

🤖 LLM API

  • POST /api/llm/chat - 普通对话
  • POST /api/llm/stream - 流式对话

⚙️ 配置 API

  • GET /api/config - 获取配置
  • PUT /api/config - 更新配置

🖼️ 图片 API

  • GET /api/images/{filename} - 获取图片
  • POST /api/images/upload - 上传图片
